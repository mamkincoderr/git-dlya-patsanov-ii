# 🛠️ Урок 1: Гит для пацанов — Установка и первый шаг

> 🎯 **TL;DR:** Ставишь Git, говоришь кто ты, создаёшь папку, пишешь `git init` — и ты уже в системе, братан.

> 💻 **Среда:** Все команды проверены на **Windows 11** с **Git 2.50.1**. Используем **PowerShell** (не cmd). Если у тебя старее — обнови Git, некоторые команды появились в версии 2.23+.

---

## 🤔 Что такое Git и зачем он тебе нужен?

Представь: ты пишешь курсовую. Делаешь копии — `курсовая_финал.docx`, `курсовая_финал2.docx`, `курсовая_точно_финал.docx`, `курсовая_ааааа.docx`.  
**Git решает эту проблему раз и навсегда.**

**Git** — это распределённая система контроля версий. Он умеет:
- 📸 Запоминать каждое изменение в коде (коммиты)
- ⏪ Возвращать код к любому предыдущему состоянию
- 🌿 Давать тебе параллельные вселенные для разных задач (ветки)
- 👥 Не убивать друг друга когда работаешь в команде

**GitHub** — это облако для твоих Git-репозиториев + социальная сеть для разработчиков. Добавляет:
- 🖥️ Красивый веб-интерфейс
- 📋 Pull Requests (твой код смотрит команда)
- 🐛 Issues (трекер задач и багов)
- ⚡ GitHub Actions (автоматизация), Pages (хостинг) и ещё кучу всего

> *Git — это инструмент на твоём компьютере. GitHub — это сервер в интернете. Это разные вещи! Можно юзать Git без GitHub, но зачем?*

## 💻 Установка Git на Windows

### Шаг 1 — Скачай
Перейди на [https://git-scm.com/download/win](https://git-scm.com/download/win) и скачай установщик.

### Шаг 2 — Установи
Запусти установщик. Везде жми **Next** с настройками по умолчанию — они нормальные.

> *Единственное что можно поменять — редактор. Если юзаешь VS Code, выбери его вместо Vim. Vim — это отдельный квест "как выйти из редактора".*

### Шаг 3 — Проверь
Открой **PowerShell** и проверь:

```powershell
git --version
# Должно написать что-то типа: git version 2.44.0.windows.1
# Если написало — ты красавчик. Если "команда не найдена" — переустанови.
```

## ⚙️ Первоначальная настройка

Git должен знать кто ты. Открой **PowerShell** и выполни:

```powershell
# Говоришь Git своё имя — появится в каждом коммите
git config --global user.name "Твоё Имя"

# Говоришь Git свой email — должен совпадать с GitHub!
git config --global user.email "твой@email.ru"

# Называем главную ветку "main" а не "master" (современный стандарт)
git config --global init.defaultBranch main

# Фикс переносов строк для Windows (очень важно в команде с маководами)
git config --global core.autocrlf true

# Открывать VS Code для написания длинных сообщений коммитов
git config --global core.editor "code --wait"
```

Проверь что всё записалось:

```powershell
git config --global --list
# Выплюнет список всех твоих настроек
```

> *`--global` значит "для всех проектов на этом компьютере". Можно делать настройки для конкретного проекта без этого флага — они перекроют глобальные.*

## 🔑 Аутентификация — без этого пуш не заработает

Когда будешь пушить на GitHub, он спросит "а ты вообще кто?". Есть два способа ответить:

### Способ 1 — Personal Access Token (PAT) — быстро и просто
1. Зайди на GitHub → **Settings** → **Developer settings** → **Personal access tokens** → **Tokens (classic)**
2. **Generate new token** → выбери срок → поставь галочку `repo`
3. **Скопируй токен** (больше не увидишь его!)
4. При первом `git push` вставь токен вместо пароля

### Способ 2 — SSH ключ — один раз и навсегда (рекомендуется)

```powershell
# Генерируем ключ (просто жми Enter на все вопросы)
ssh-keygen -t ed25519 -C "твой@email.ru"

# Копируем публичный ключ в буфер обмена (команда для PowerShell на Windows)
Get-Content "$env:USERPROFILE\.ssh\id_ed25519.pub" | clip
# Или просто выведи и скопируй руками:
Get-Content "$env:USERPROFILE\.ssh\id_ed25519.pub"
```

Потом на GitHub → **Settings** → **SSH and GPG keys** → **New SSH key** → вставляй.

> ⚠️ **Windows: не забудь запустить ssh-agent!** Без этого ключ создан, но не работает:

```powershell
# Запускаем ssh-agent (один раз, можно добавить в профиль PowerShell)
Set-Service -Name ssh-agent -StartupType Manual
Start-Service ssh-agent

# Добавляем ключ
ssh-add "$env:USERPROFILE\.ssh\id_ed25519"

# Проверяем что всё работает
ssh -T git@github.com
# Должно сказать: "Hi ИМЯ! You've successfully authenticated..."
```

> *SSH — это как пропуск. Один раз показал охраннику, дальше просто проходишь. PAT — это как каждый раз предъявлять паспорт. Оба работают, SSH удобнее.*

## 🏗️ Создание первого репозитория

```powershell
# Создаём папку проекта
mkdir git-practice
cd git-practice

# Инициализируем Git — создаётся скрытая папка .git
git init
# Git скажет: "Initialized empty Git repository"
# Поздравляю, ты только что создал репу!

# Смотрим что происходит
git status
# Скажет что нет коммитов и нечего трекать — это нормально, ты только начал
```

Теперь создай первый файл и сделай первый коммит:

```powershell
# Создаём README.md
New-Item README.md
Add-Content README.md "# Мой первый репозиторий"

# Добавляем файл в индекс
git add README.md

# Делаем первый коммит
git commit -m "feat: initial commit"
# Git скажет: "[main (root-commit) abc1234] feat: initial commit"
# Это твой первый снимок истории. Поздравляю!
```

> *Папка `.git` — это мозг репозитория. Не удаляй её. Если удалишь — Git забудет всю историю и станет обычной папкой.*

## 💀 Типичные ошибки новичка

| Ошибка | Что происходит |
|--------|---------------|
| Не настроил email перед первым коммитом | Коммиты без автора, потом не разберёшься |
| Email не совпадает с GitHub | Коммиты не привязываются к твоему профилю |
| Удалил папку `.git` | Репозиторий стал обычной папкой, история потеряна |
| Не настроил `autocrlf true` | Коллеги на Mac/Linux видят изменения в каждой строке каждого файла |
| Запушил без токена/SSH | Получил ошибку аутентификации и не понял почему |
