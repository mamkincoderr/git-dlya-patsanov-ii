# 🖥️ Бонус-урок: Git в VSCode — всё то же самое, но кликами

> 🎯 **TL;DR:** Всё что ты делал в терминале — `add`, `commit`, `diff`, ветки, конфликты — VSCode умеет делать мышкой. Это не замена терминалу, это второй способ делать то же самое.

---

Ты уже знаешь команды. Ты уже коммитил через терминал и не умер. Теперь секрет который не говорят вслух:

**Большинство разработчиков делают рутинный Git мышкой прямо в редакторе.** А терминал открывают только для сложных вещей — rebase, cherry-pick, разруливание серьёзных конфликтов.

VSCode умеет в Git из коробки — без плагинов, без настройки. Просто открываешь папку с репозиторием — и всё работает.

---

## 📍 Где это всё живёт — Source Control панель

Нажми **Ctrl+Shift+G** (или кликни на иконку веток в левом сайдбаре):

```
┌─────────────────────────────────────────┐
│ ⎇  SOURCE CONTROL           ⟳  +  ✓  … │
├─────────────────────────────────────────┤
│                                         │
│   Сообщение коммита...                  │
│                                         │
│                           [Commit ▾]    │
│                                         │
├─── CHANGES (3) ─────────────────────────┤
│                                         │
│  M  src/app.js                      +   │
│  M  style.css                       +   │
│  ?  notes.txt                       +   │
│                                         │
└─────────────────────────────────────────┘
```

Что видишь:
- **CHANGES** — файлы которые изменились, но ещё не добавлены в индекс (`git status` красные)
- **M** — Modified, изменён
- **?** — Untracked, Git вообще не знает про этот файл
- **+** справа — кнопка "добавить в индекс" (`git add` для одного файла)
- **⟳** — обновить статус
- **+** вверху — добавить все файлы сразу (`git add .`)
- **✓** — закоммитить всё что в индексе

## ➕ Stage — добавляем файлы в индекс

В терминале ты делал `git add файл.txt`. В VSCode — наводишь на файл и жмёшь **+**:

```
┌─── CHANGES (3) ─────────────────────────┐
│                                         │
│  M  src/app.js               [+]  [↺]  │  ← навёл мышку
│  M  style.css                           │
│  ?  notes.txt                           │
│                                         │
├─── STAGED CHANGES (1) ──────────────────┤
│                                         │
│  M  src/app.js               [-]  [↺]  │  ← файл попал сюда
│                                         │
└─────────────────────────────────────────┘
```

Видишь что произошло: `app.js` переехал из **CHANGES** в **STAGED CHANGES**. Это и есть индекс — зона ожидания перед коммитом.

**[-]** рядом — убрать файл обратно из индекса (`git restore --staged`).

> *Ты можешь добавить только `app.js`, а `style.css` оставить на потом. Именно за это и нужен двухшаговый процесс — один коммит про одно изменение, не всё подряд.*

## 📸 Commit — коммитим через интерфейс

Написал сообщение коммита в поле сверху, нажал **Commit**:

```
┌─────────────────────────────────────────┐
│                                         │
│   feat: добавил кнопку авторизации      │  ← пишешь сюда
│                                         │
│                           [Commit ▾]    │  ← жмёшь сюда
│                                         │
└─────────────────────────────────────────┘
```

Стрелка ▾ рядом с кнопкой — выпадает меню с вариантами:
- **Commit** — просто закоммитить
- **Commit & Push** — закоммитить и сразу запушить
- **Commit & Sync** — закоммитить, подтянуть чужие изменения, запушить

> *"Commit & Sync" удобен в командной работе — делает за тебя `pull` + `push` одним кликом. Но если есть конфликты, придётся разруливать.*

## 🔍 Diff — смотрим что изменилось

Кликни на любой файл в списке Changes — откроется diff прямо в редакторе:

```
┌──────────────────┬──────────────────────┐
│   БЫЛО           │   СТАЛО              │
├──────────────────┼──────────────────────┤
│                  │                      │
│  function greet()│  function greet()    │
│    return "hi"   │    return "привет"   │  ← зелёный
│                  │    console.log("ok") │  ← зелёный (добавлено)
│  }               │  }                   │
│                  │                      │
│  const x = 1    │                      │  ← красный (удалено)
│                  │                      │
└──────────────────┴──────────────────────┘
```

Слева — было, справа — стало. Зелёный — добавлено, красный — удалено. Это то же самое что `git diff` в терминале, только глазами видно сразу.

> *Смотри diff перед каждым коммитом. Это займёт 10 секунд и спасёт тебя от "ой, я случайно закоммитил консоль.лог в продакшн".*

## 🌿 Ветки — переключаемся без терминала

В самом низу VSCode в статус-баре видна текущая ветка:

```
┌────────────────────────────────────────────────────────┐
│  ⎇ main    ✓ 0↓ 2↑    Ln 42, Col 8    UTF-8    ...    │
└────────────────────────────────────────────────────────┘
```

- **⎇ main** — текущая ветка, кликни чтобы переключиться
- **0↓** — сколько коммитов нужно подтянуть с сервера
- **2↑** — сколько твоих коммитов ждут пуша

Кликаешь на имя ветки — открывается меню:

```
  ──────────────────────────────
  + Create new branch...
  + Create new branch from...
  ──────────────────────────────
  ⎇ main
  ⎇ book-improvements          ← переключиться одним кликом
  ──────────────────────────────
```

## ⚔️ Конфликты — это не страшно когда видишь их глазами

Конфликт в терминале выглядит примерно так:

```
<<<<<<< HEAD
кнопка синяя
=======
кнопка красная
>>>>>>> feature/button-color
```

Три куска непонятного текста, разберись сам. Новичок в ужасе.

В VSCode та же ситуация выглядит по-человечески:

```
┌──────────────────────────────────────────────────────┐
│  [Принять текущее]  [Принять входящее]  [Оба]        │
├──────────────────────────────────────────────────────┤
│                                                      │
│  <<<< CURRENT (HEAD)                                 │
│  кнопка синяя                                        │
│  ════════════════                                    │
│  кнопка красная                                      │
│  >>>> INCOMING (feature/button-color)                │
│                                                      │
└──────────────────────────────────────────────────────┘
```

Три кнопки вверху:
- **Принять текущее** — оставить твоё изменение
- **Принять входящее** — взять чужое изменение
- **Оба** — оставить и то и то
- Или просто вручную отредактировать как тебе нужно

> *Большинство конфликтов решаются за 30 секунд когда видишь их вот так. В терминале та же операция занимает 5 минут нервных поисков тегов `<<<`.*

## 🔌 Расширения которые реально нужны

Одно — обязательно, остальные по желанию:

**GitLens** — суперсила для работы с Git в VSCode.
Показывает прямо в коде кто и когда изменил каждую строку, полную историю файла, сравнение веток. Ставится из маркетплейса (Ctrl+Shift+X → поиск "GitLens").

```
│  42  │  const color = "blue"   ← Иванов, 3 дня назад: feat: цвет кнопки
```

> *Это называется git blame — "обвинить" кого-то в этой строке кода. С GitLens это видно везде и всегда, без отдельной команды.*

## 💀 Типичные ошибки

| Ошибка | Что происходит |
|--------|----------------|
| Нажал "Commit" без сообщения | VSCode ругается, коммита нет — хорошо что не пропустил |
| Нажал "Commit & Sync" с конфликтами | VSCode остановится и покажет конфликты, не страшно |
| Не смотришь diff перед коммитом | Коммитишь лишнее: дебаг-код, console.log, случайные изменения |
| Думаешь что VSCode-кнопки — это что-то другое | Это буквально те же git-команды, просто без терминала |

---

> **Важно:** VSCode не заменяет знание команд — он их прячет за кнопками. Поэтому ты сначала учил терминал. Теперь ты понимаешь что происходит за каждым кликом — это и есть разница между "нажал кнопку" и "знаю что делаю".
