# 🌐 Урок 3: Подключаем GitHub — выходим в интернет

> 🎯 **TL;DR:** GitHub — это облако для твоих репозиториев. Создаёшь аккаунт, настраиваешь аутентификацию, делаешь `git push` — и твой код живёт в интернете.

---

В прошлом уроке ты научился работать с Git локально. Всё это время история коммитов жила только на твоём компьютере. Теперь поднимаем её в облако.

**Зачем нужен GitHub:**
- 💾 Резервная копия — жёсткий диск умрёт, код выживет
- 👥 Командная работа — общее место куда все пушат
- 🚀 CI/CD и деплой — автоматическая сборка и публикация
- 🌍 Портфолио — работодатели смотрят твой GitHub

> *GitHub — не единственный вариант. Есть GitLab (популярен в корпоративной среде, есть self-hosted), Gitea (поднимаешь на своём сервере за 10 минут, полный контроль). Но для начала — GitHub.*

## 👤 Создаём аккаунт

Зайди на [https://github.com](https://github.com) и зарегистрируйся. Никаких хитростей — имя, email, пароль.

> *Выбирай username осторожно — он будет в ссылках на твои проекты и в резюме. `mamkincoderr` — норм. `xxx_darkSOUL_2007_xxx` — возможно, не лучший выбор для работодателя.*

## 🔑 Аутентификация — без этого пуш не заработает

Когда делаешь `git push`, GitHub спросит "а ты вообще кто?". Есть два способа ответить:

### Способ 1 — Personal Access Token (PAT) — быстро и просто

1. GitHub → **Settings** → **Developer settings** → **Personal access tokens** → **Tokens (classic)**
2. **Generate new token** → выбери срок действия → поставь галочку `repo`
3. **Скопируй токен сразу** — больше не увидишь его
4. При первом `git push` вставь токен вместо пароля

> *PAT — как разовый пропуск. Удобно для быстрого старта. Но если токен утечёт — злоумышленник получит доступ к репе. Храни его как пароль.*

### Способ 2 — SSH ключ — один раз и навсегда (рекомендуется)

SSH — это криптографический ключ. Генерируешь пару: приватный ключ остаётся у тебя, публичный отдаёшь GitHub. При подключении GitHub проверяет пару — и пускает без пароля.

```powershell
# Генерируем ключ (жми Enter на все вопросы — оставляй путь и пароль по умолчанию)
ssh-keygen -t ed25519 -C "твой@email.ru"

# Выводим публичный ключ — его нужно скопировать
Get-Content "$env:USERPROFILE\.ssh\id_ed25519.pub"
```

Скопируй весь вывод (начинается с `ssh-ed25519`, заканчивается email).

Потом на GitHub: **Settings** → **SSH and GPG keys** → **New SSH key** → вставляй и сохраняй.

Запускаем ssh-agent и добавляем ключ:

```powershell
# Запускаем ssh-agent (делается один раз)
Set-Service -Name ssh-agent -StartupType Manual
Start-Service ssh-agent

# Добавляем ключ
ssh-add "$env:USERPROFILE\.ssh\id_ed25519"

# Проверяем что всё работает
ssh -T git@github.com
# Должно сказать: "Hi ИМЯ! You've successfully authenticated..."
```

> *SSH — как пропуск на проходной. Один раз оформил, дальше просто проходишь. PAT — как каждый раз предъявлять паспорт. Оба работают, SSH удобнее в долгосрочной перспективе.*

## 🚀 Пушим первый репозиторий на GitHub

Создай новую репу на GitHub: кнопка **New** → дай имя (например `git-practice`) → **Create repository**. Не добавляй README, .gitignore — у тебя уже есть локальная история.

GitHub сразу покажет команды. Нам нужен второй блок — "push an existing repository":

```powershell
# Говоришь Git куда пушить (origin — стандартное имя удалённого репозитория)
# Если используешь SSH:
git remote add origin git@github.com:ТВО_ИМЯUSERNAME/git-practice.git

# Если используешь PAT (HTTPS):
git remote add origin https://github.com/ТВОЙ_USERNAME/git-practice.git

# Пушим ветку main на GitHub (-u запоминает куда пушить, в следующий раз просто git push)
git push -u origin main
```

Зайди на GitHub — там уже твои файлы и история коммитов.

```powershell
# Проверить что remote настроен правильно
git remote -v
# Покажет: origin  git@github.com:USERNAME/git-practice.git (fetch)
#          origin  git@github.com:USERNAME/git-practice.git (push)
```

## 💀 Типичные ошибки новичка

| Ошибка | Что происходит |
|--------|---------------|
| Email не совпадает с GitHub | Коммиты не привязываются к твоему профилю на GitHub |
| Скопировал HTTPS-ссылку и настроил SSH | `git push` требует пароль каждый раз |
| Потерял PAT-токен | Придётся генерировать новый — GitHub не показывает токен повторно |
| Добавил `--bare` в `git init` | Создал голый репозиторий, файлы не видны — удали и пересоздай без флага |
| Запушил `.env` с паролями в публичную репу | Меняй все ключи срочно — GitHub-боты сканируют за минуты |
